## 第三步：合并四组结果

将step01a、step01b、step02a和step02b的表格数据合并为一张大表：
- 保留所有列（标题、作者、期刊、发表日期、DOI、发表状态、摘要、论文关键词、核心发现、检索来源、关键词编号）
- 新增「检索组」列，标注来源（A1/A2/B1/B2）
- 合并时保留完整摘要，不做截取

**记录合并统计**：记录每个来源组的论文数量

---

## 第四步：跨组去重

**主键**：DOI

**规则**：
- 若两行DOI相同 → 保留信息更完整的那行
- 合并「检索来源」「关键词编号」「检索组」列（取并集，用`; `分隔）
- 例：同一论文被组A1（#1,#3）和组B2（#34）同时召回 → 合并后关键词编号为 `#1,#3,#34`，检索组为 `A1;B2`

**无DOI时的备选去重**：
- 基于「标题 + 第一作者 + 发表年份」判断
- 标题相似度 ≥90% 视为重复
- 预印本和已发表版本：若标题相同且作者高度重合，视为重复，保留已发表版本

**核心发现合并**：若同一论文在不同检索组中生成了不同的核心发现描述，保留更详细完整的版本。

**去重后一致性抽检**：
- 从去重后的结果中随机抽取5条（不足5条则全部抽）
- 逐条检查：标题是否与摘要内容相关
- 若发现标题与摘要明显不匹配 → 在该条记录「备注」列标注"⚠️ 标题-摘要不一致"

## 第五步：调用文献相关性校验技能

使用 Skill 工具调用 `文献相关性校验`：

### 5.1 准备输入数据

从合并去重后的文献列表中提取：

```json
{
  "papers": [
    {
      "id": "序号",
      "title": "论文标题",
      "abstract": "论文摘要",
      "keywords": "论文关键词",
      "source": "检索来源",
      "keyword_ids": ["关键词编号"]
    }
  ],
  "options": {
    "threshold": 0.2,
    "output_detail": true
  }
}
```

### 5.2 执行校验

调用技能后，获取校验结果（JSON格式）。

### 5.3 处理校验结果

根据校验结果更新文献列表：

| 相关性分类 | 处理方式 |
|------------|----------|
| 强相关 (≥0.7) | 直接收录 |
| 弱相关 (0.4-0.7) | 收录，标记需巡检 |
| 待确认 (0.2-0.4) | 收录，标记需人工确认 |
| 不相关 (<0.2) | 排除，标注原因 |

### 5.4 更新去重后的文献列表

在原表格基础上新增列：
- `相关性评分`：0-1的数值
- `相关性分类`：强相关/弱相关/待确认/不相关
- `匹配方向`：命中的方向编号
- `匹配对象`：命中的研究对象
- `处理建议`：直接收录/需巡检/需确认/建议排除

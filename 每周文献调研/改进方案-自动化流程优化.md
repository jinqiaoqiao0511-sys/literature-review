# 每周文献调研 — 自动化流程改进方案

---

## 问题一：缺少执行前校验机制

### 问题描述
- 流程直接运行，没有对检索到的新文章进行前置校验
- 可能检索到不相关的文献也被收录，导致文献库质量下降

### 改进方案：新增 auto-00 预检任务

**任务定位**：在所有检索任务之前执行，判断本周是否有必要执行检索

**触发条件**：
```
每周一 00:00 执行（在 auto-01a 之前）
```

**检查内容**：
1. **学术淡季检测**：
   - 检查过去 4 周的文献命中数量趋势
   - 若连续 2 周命中数 < 5 篇，触发警告

2. **关键词覆盖度检测**：
   - 检查核心方向（#5, #10, #11, #12, #13）的命中情况
   - 若连续 3 周某核心方向零命中，建议扩展关键词

3. **检索工具可用性检测**：
   - 测试 ArXiv、WebSearch、Tavily 等工具的连通性
   - 若主要工具不可用，推迟执行并通知

**输出文件**：`中间结果/step00-preflight-YYYY-WXX.md`

```markdown
# 预检报告 - YYYY-WXX

## 检测结果

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 学术淡季检测 | ✅/⚠️ | [说明] |
| 关键词覆盖度 | ✅/⚠️ | [说明] |
| 工具可用性 | ✅/❌ | [说明] |

## 建议
- [具体建议]

## 决策
- [ ] 继续执行（默认）
- [ ] 推迟执行
- [ ] 扩展关键词后执行
```

## 问题二：流程"强行运行"导致收录不相关文献

### 问题描述
- 当文献库积累到一定程度，没有新的可扩充内容时，流程仍要执行
- 系统会找一些不相关的文献来"凑数"，降低了文献相关性

### 改进方案：相关性校验机制

**校验位置**：在 auto-03a（数据整合）阶段增加相关性校验

**校验规则**：
```markdown
## 相关性校验标准

### 强相关（直接收录）
- 标题包含研究方向关键词
- 摘要明确涉及研究主题
- 来自核心期刊（第一/二梯队）

### 弱相关（标注后收录）
- 间接相关（如方法论、综述）
- 仅关键词编号命中，但内容偏离
- 来自边缘期刊

### 不相关（标记排除）
- 与研究方向无明显关联
- 仅因关键词歧义被检索到
```

**实现方式**：
1. 在 step03-classified 中新增 `相关性评级` 列
2. 评级标准：
   - `强相关`：与核心方向直接相关
   - `弱相关`：间接相关，有参考价值
   - `待确认`：需要人工复核
   - `不相关`：建议排除（不删除，仅标注）

3. 自动标注逻辑：
   ```
   IF 核心方向命中 AND 高优期刊 → 强相关
   IF 核心方向命中 AND 普通期刊 → 弱相关
   IF 非核心方向 AND 关键词编号命中 → 待确认
   IF 无明确关联 → 不相关
   ```

## 问题三：缺少文献质量标注机制

### 问题描述
- 对已收录文献没有区分"强相关"和"弱相关"
- 无法快速识别哪些文献值得深入阅读

### 改进方案：建立文献标注体系

**新增字段**（在 step03-classified 和 Obsidian 笔记中）：

```markdown
## 文献标注字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 相关性评级 | 枚举 | 强相关/弱相关/待确认/不相关 |
| 阅读状态 | 枚举 | 未读/已读/精读/已标注 |
| 人工标注 | 文本 | 用户自定义标签（如"方法论"、"综述"、"实验数据"） |
| 阅读笔记 | 长文本 | 自动摘要 + 人工补充 |
| 复核日期 | 日期 | 最后一次人工复核时间 |
```

**标注工作流**：
1. **自动标注**（auto-03b）：
   - 根据相关性校验规则自动标注
   - 生成初始阅读状态

2. **人工复核**（巡检时）：
   - 逐篇检查相关性评级
   - 补充阅读笔记
   - 修正不准确的标注

3. **长期积累**：
   - 建立"文献画像"，识别用户真正关心的研究方向
   - 优化检索关键词

## 问题四：缓存文件信息不完整

### 问题描述
- 更新缓存时只有基本字段（期刊名、JCR分区、影响因子）
- 缺少文献内部具体信息（方法、数据、结论）

### 改进方案：深度阅读补充缓存

**调用技能**：`arxiv-reader`（已有三档深度：裸读、引导、精读）

**新增任务**：auto-06 文献深度阅读

**执行时机**：
```
每周一 10:00 执行（在巡检之后）
或手动触发（对特定文献）
```

**任务流程**：
```markdown
## auto-06 任务流程

### 输入
- 本周 step03-classified 中评级为 ⭐⭐⭐⭐⭐ 的文献
- 用户手动指定的文献

### 处理步骤
1. **获取论文全文**
   - ArXiv 预印本 → 直接获取
   - 已发表论文 → 尝试获取摘要 + 图表

2. **调用文献阅读技能**
   - 裸读：提取基本信息（标题、作者、摘要）
   - 引导：提取方法、结果、结论
   - 精读：提取创新点、局限性、与其他研究对比

3. **更新缓存文件**
   - 在原有缓存基础上补充：
     - 研究方法摘要
     - 关键数据/图表说明
     - 创新点总结
     - 与其他研究的关联

### 输出
- 更新 `文献调研/2026-WXX/001-xxx.md` 笔记
- 生成 `中间结果/deep-reading-YYYY-WXX.md` 汇总
```

**缓存文件扩展格式**：
```markdown
# 文献缓存 - [DOI]

## 基本信息（原有）
| 字段 | 值 |
|------|---|
| 期刊名 | [期刊名] |
| JCR分区 | [Q1/Q2/Q3/Q4] |
| 影响因子 | [数值] |
| 更新日期 | [YYYY-MM-DD] |

## 深度阅读（新增）
| 字段 | 值 |
|------|---|
| 阅读深度 | 裸读/引导/精读 |
| 研究方法 | [方法摘要] |
| 关键数据 | [重要数据/图表] |
| 创新点 | [创新点总结] |
| 局限性 | [研究局限] |
| 关联研究 | [与其他研究的关联] |
| 阅读日期 | [YYYY-MM-DD] |
```

## 问题五：自动化程度不足

### 问题描述
- 需要人工阅读文献内容
- 调研时无法直接调用缓存内容进行总结

### 改进方案：构建文献知识库

**目标**：实现"调研 → 阅读 → 总结"全自动化

**架构设计**：
```
┌─────────────────────────────────────────────────────────┐
│                    文献知识库架构                         │
├─────────────────────────────────────────────────────────┤
│  检索层 (auto-01/02)                                     │
│    ↓                                                     │
│  整合层 (auto-03)                                        │
│    ↓                                                     │
│  阅读层 (auto-06) ← 新增                                 │
│    ↓                                                     │
│  知识层 (auto-07) ← 新增                                 │
│    ↓                                                     │
│  输出层 (auto-04/05)                                     │
└─────────────────────────────────────────────────────────┘
```

**新增任务**：auto-07 知识库构建

**任务内容**：
```markdown
## auto-07 知识库构建

### 功能
1. **文献关联分析**
   - 识别同一研究方向的文献群
   - 发现研究趋势和热点

2. **知识图谱构建**
   - 提取文献中的关键概念
   - 建立概念之间的关联

3. **智能问答**
   - 基于知识库回答用户问题
   - 生成研究方向综述

### 输出
- `知识库/concept-graph.json` - 概念图谱
- `知识库/research-trends.md` - 研究趋势
- `知识库/QA-pairs.md` - 问答对
```
